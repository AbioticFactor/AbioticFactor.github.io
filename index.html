Lab 2: Bluetooth

The goal of this lab is to connect our Artemis board to a computer through bluetooth and learn how the protocall works with the artemis through the implemtnation of various functions.

Hardware
  • Bluetooth Adapter: I used the provided bluetooth adapter to enable bluetooth connectivity on the lab computer.
Software
  • ArduinoBLE: One of the main components of this lab was the ArduinoBLE library. This library enables us to use Bluetooth® Low energy on our Artemis. Bluetooth low energy works through a client/ server model where a device initiates commands and requests and another one recieves them. The data sent between them are called "characteristics" which can include things like a sensor reading or a voltage. The library page can be viewed here
  • Codebase: Another main component of this lab was the provided codebase. The codebase includes functions to send and recieve various "robot commands". These are functions defined by the format “cmd_type:value1|value2|value3|…”. The data is handled by various constructors for different characteristics. The code base can be downloaded here
  • Jupyter Lab: A big part of this lab was the creation fo Jupyter Notebooks through Jupyer Lab. . Jupyter lab allowed us to write and test interactive python scripts over the course of the lab. The official site for Jupyter Lab may be found here

Limitations

While the artemis is a pretty cool chip, it is heavily limited by the amount of data it can send over. Lets say for example you wanted to store 5 seconds of 16 bit values taken at 150hz. This would be equivalenbt to 2400 bits every second. The Artemis only contains 384kb of ram which is equivalent to 3072000 bits. At the rate given we would only be able to send 256 of these 5 second groups, thats not that much!

Software
  • I2C : The main communication protocall we will be looking at this lab is Inter-Integrated Circuit communications (I2C). I2C is special in that it has features from both SPI and UART (Other communication protocalls) and it only uses two wires to communicate. Serial Data (SDA) sends and recieves data while Serial Clock (SCL) carries the clock signal. The default I2C address for our VL53L1X should be 0x52 but upon scanning our ports we see that it is 0x29. This is because the address is bit shifted. The library page can be viewed here
